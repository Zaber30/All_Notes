By Function (What They Do)

- **Web Server:** Delivers web pages (HTML, images) to browsers. Examples: Nginx, Apache.
- **Application Server:** Runs the core business logic and dynamic code of an app. Examples: Node.js, Tomcat.
- **Database Server:** Stores, manages, and retrieves structured data. Examples: PostgreSQL, MySQL, MongoDB.
- **File Server:** Stores and shares files across a network. Examples: Network Attached Storage (NAS), FTP servers.
- **Mail Server:** Handles, routes, and stores emails. Examples: Postfix, Microsoft Exchange.
- **DNS Server:** Translates human-readable domain names (like google.com) into computer IP addresses.
- **Proxy / Reverse Proxy Server:** Sits between users and web servers to improve security, speed, and load balancing

## Server Types by Infrastructure (How They Are Built)

### 1. Bare Metal / Dedicated Server

A **bare metal server** is a physical computer machine where the entire hardware is dedicated to a single customer or organization.

### 2. Virtual Private Server (VPS)

A **VPS** is a virtual server created by dividing one physical server into multiple isolated virtual machines using virtualization technology.

### 3. Cloud Server

A **cloud server** is a virtual server created and managed through a cloud computing platform. It runs across a large network of physical servers in data centers.

Examples:

- Amazon Web Services EC2
- Google Cloud Compute Engine
- Microsoft Azure Virtual Machines
### 4. Edge Server

An **edge server** is a server placed physically close to users to reduce delay (latency) and improve performance.
